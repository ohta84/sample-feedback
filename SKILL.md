# サンプルフィードバックExcel自動生成

Bカートの受注CSV（1本、複数可）を受け取り、メーカー別のサンプルフィードバックExcelを自動生成するスキル。

## 必要なファイル

**Bカート受注CSV（1本、または複数）**（例: `bcart_order.csv`）

Bカートの管理画面は一度に500件までしかCSV出力できない制限があるため、
件数が多いメーカーでは複数回に分けてエクスポートされたCSVが複数本届くことがある。
その場合は全ファイルを受け取り、`--order-csv file1.csv file2.csv ...` のように
スペース区切りで複数指定する（内部で結合し、完全重複行は自動的に除去する）。

受注CSVには以下がすべて含まれているため、これ1本で完結する：
- 会社名・納品日・商品名・業態・配送先都道府県・受注番号
- **セット名**（サンプル注文か通常注文かの判定に使う）
- **サンプル用途・サンプル評価結果・評価都合／採用数**（サンプルの評価コメント）
- 受注数・小計（受注データ分析シート用。ただし小計＝金額は使わない）

「サンプル評価回収CSV」や「サンプル評価結果CSV」といった評価データだけの抜粋CSVは、
受注CSVから作られた**サブセット（部分集合）**でしかなく、追加の情報は持っていない。
ユーザーがこれらを渡してきた場合は、受注CSV（全項目版）を渡すよう伝える。
万一受注CSVを用意できず評価データCSVしか無い場合は、シート1（サンプル評価結果の一覧）は
作れるが、注文転換率やシート3（受注データ分析）は正しく算出できない旨を必ず伝えること。

## 全体の流れ

1. ユーザーから受注CSV（1本、複数に分かれている場合は全ファイル）を受け取る
2. 会社名に「ナゴミヤ」「なごみや」を含む行は自社テストのため除外
3. **メーカーコードはユーザーに聞かず、商品名冒頭（`[121]` 等）から自動検出する**（1メーカー分のCSVが前提。複数コードが混在する場合のみ、最頻出コードを採用しつつユーザーに確認する）
4. メーカー名は `data/maker_codes.csv`（コード→会社名の対応表）から自動解決する。解決した会社名から「株式会社」「有限会社」などの法人格は取り除く（例: コード138・「株式会社アンデ」→ タイトル/ファイル名は「138アンデ様サンプルフィードバック」）
5. セット名に「サンプル」を含む行＝サンプル提供、含まない行＝通常注文として分類する
6. 会社ごとに、サンプル提供後に通常注文（転換）が1件でもあるかを同じCSV内で判定する
7. `scripts/generate_feedback_excel.py` を実行してExcelを生成する（`--maker-code`・`--maker-name`・`--output` は省略可。省略時は自動検出・自動命名される）
8. 生成後、社数・転換率などのサマリーをユーザーに提示し、内容に問題ないか確認する

ユーザーは基本的に受注CSV1本を渡すだけでよく、メーカー名やコードを毎回聞く必要はない。
ただし、コード表に無いメーカーコードが検出された場合はユーザーに会社名を確認し、`data/maker_codes.csv` に追記する。

## メーカーコード対応表（data/maker_codes.csv）

「数字, 会社名」の2列CSV。商品名冒頭の `[141]` のようなコードとメーカー名の対応表。
新しいメーカーコードが出てきたら、ユーザーに確認してこのファイルに追記していく。

`--maker-name` を省略して `--maker-code` だけ指定すると、この表から自動的に
`{コード}{会社名}` という形式（例: `141東京堂`）でタイトル・ファイル名が決まる。

## Excel生成ロジック（scripts/generate_feedback_excel.py に実装済み）

### シート1: サンプルフィードバック

サマリー（行1〜6）
- 行1: タイトル「(コード)(メーカー名)様サンプルフィードバック」太字・サイズ18・折り返しなし
- 行2: サンプル提供社数（ユニーク会社数）
- 行3: 注文転換社数（サンプル提供後に通常注文を1件以上行った会社数）
- 行4: 注文転換率＝転換社数÷提供社数。赤字・太字・%表示
- 行5: 空白
- 行6: 左＝納品日の期間（最小〜最大）、右＝生成日（例: 2026/7/13）＋全角スペース＋「なごみや太田」（作成者名・変更しないこと）

ヘッダー行（行7）とデータ（行8〜）

**データの並び順：納品日の昇順**（同じ日付の場合は会社名順）。会社名順ではなく日付順であることに注意。

| 列 | 内容 | 幅 |
|---|---|---|
| A | 納品日（YYYY/MM/DD） | 14.6 |
| B | 業態 | 15.1 |
| C | 会社名 | 31.6 |
| D | 配送先都道府県 | 11.9 |
| E | 商品名 | 37.0 |
| F | サンプル用途 | 35.2 |
| G | サンプル評価結果（評価都合／採用数＋サンプル評価結果） | 71.4 |
| H | 注文有無 | 14.0 |

- ヘッダー行はグレー背景・中央寄せ・折り返しあり・枠線あり
- 同じ会社・同じ納品日の行が複数商品にまたがる場合、A,B,C,D,F,G,H 列をセル結合（E商品名のみ全行に入力）
- 注文有無：「✅ 注文あり」緑背景（E2EFDA）・緑文字（375623）・太字／「－ なし」グレー文字（999999）
- フォントは全体 Arial・サイズ11、枠線は薄いグレー（BFBFBF）細線
- セルの縦位置は基本的にすべて中央揃え（タイトル行など単一行の見出しを除く）。
  **注意：ExcelJS（Web版）では縦方向の指定に `"center"` ではなく `"middle"` を使うこと**
  （`"center"` を指定すると出力XMLに反映されず、意図せず既定値になってしまう）。
  openpyxl（Python版）では通常通り `vertical="center"` でよい。

### シート2: 業態別分析

業態ごとに以下を集計する表（提供社数の多い順にソート）：
- 提供社数／転換社数／転換率
- OK件数・NG件数・検討中件数

**年ごとの自動分割（横並び）**：サンプル納品日に複数の年（例: 2024年・2025年・2026年）が
混在する場合、年ごとに別々の表として**横方向に並べて**出力する（「2024年」「2025年」…の見出しを
付けた表がA列, I列, Q列...のように横に展開される。ブロック間は1列の空白列を挟む）。
同じ会社が複数年にまたがってサンプル提供を受けている場合は、それぞれの年で1件としてカウントする
（転換有無の判定は全期間で見る）。年が1つしか無い場合は従来通り見出し無しの単一表のまま
出力する（無用な複雑化を避ける）。

**OK/NG/検討中の判定ロジック**（受注番号ごとに1回だけ判定し、複数商品行での重複カウントを避ける）：
1. サンプル評価結果の文章内にOK/NG/検討中の明示的な表記がある場合 → その出現回数をそのまま使う
2. 上記が無く、「評価都合／採用数」がスラッシュを含まない数字の場合 → その数字＝OK（採用数）、
   （同一受注番号内の商品行数－採用数）＝NG
3. 「7/8」のようにスラッシュを含む値は日付の書き間違い（古いデータに見られる）なので無視し、4に進む
4. どちらも無い自由記述のみの場合 → 「採用」「注文します」「注文したい」「発注」「導入」「欲しい」
   「希望」などの語があればOK、「不採用」「見送り」「お断り」「断念」などの語があればNG、
   どちらも無ければ検討中（該当する1区分のみ+1し、他は0）

`scripts/generate_feedback_excel.py` の `classify_evaluation()` に実装済み。ロジックを変更する場合は
Python版・Web版（index.html内の`classifyEvaluation`）の両方を同じ内容に保つこと。

### シート3: 受注データ分析

同じ受注CSVの通常注文行のみを対象に集計する：

- 表1「業態別 受注動向」：業態ごとの受注社数・受注件数・受注数量合計（受注件数順にソート）
- 表2「商品別ランキング」：商品ごとの受注数量合計・購入社数（数量上位10件）
- 表3「会社別ランキング」：会社名・業態・受注数量合計・受注件数（数量上位10社）

「受注数」列から算出する。サンプル注文（商品代無料）は除外して集計すること。

**年ごとの自動分割（横並び）**：シート2と同様、通常注文の納品日に複数の年が混在する場合は
年ごとに分けて出力する。ただし縦積みではなく、**セクション（業態別受注動向／商品別ランキング／
会社別ランキング）ごとに、年を横方向に並べる**レイアウトにする：
1行目に業態別受注動向を年数分横に並べ、2ブロック分下に商品別ランキングを年数分横に並べ、
さらに下に会社別ランキングを年数分横に並べる、という構成（`write_one_table`/`writeOneTable`で
`col_offset`／`colOffset`を各年ブロックごとにずらして書き込む）。年が1つしか無い場合は
見出し無しの単一セットのまま出力する。

**重要：金額（単価・小計）は絶対に含めないこと。** 受注CSVの「単価」「小計」はなごみやが
店舗に販売する価格（卸値）であり、メーカーへの仕入価格とは異なる。メーカー向け資料に金額を
載せると、メーカー自身が知っている仕入価格と比較されてなごみやの粗利率が算出できてしまうため、
このシートは数量・件数・社数のみで構成する。

## 実行方法

```bash
pip install pandas openpyxl --break-system-packages
python scripts/generate_feedback_excel.py --order-csv "bcart_order.csv"
```

CSVが1メーカー分に絞られていれば、これだけでメーカーコード検出・メーカー名解決・
ファイル名（`{コード}{会社名}様サンプルフィードバック.xlsx`）まで自動で完結する。

`--maker-name` / `--maker-code` / `--output` はいずれも任意。複数メーカーが混在するCSVを
特定コードだけに絞りたい場合や、コード表に無いメーカーを扱う場合にのみ指定する。

生成後は必ず `present_files` でユーザーに提示すること。

## 注意事項

- スクリプトの実行はClaude上で完結するので、ユーザー側のインストール作業は不要
- 「なごみや太田」の部分は作成者名なので絶対に変更しない
- CSVの実際のヘッダー名がBカートの標準と異なる場合があるため、スクリプトは列名を柔軟にキーワード一致で探す。うまく検出できない場合はエラーメッセージで該当箇所を出す
- 新しいメーカーコードが出てきたら `data/maker_codes.csv` に追記して更新する
- ユーザーから「評価データCSVだけ」「評価回収CSVだけ」等を渡された場合は、受注CSV（全項目版）が
  必要であることを伝え、代わりを求める（評価データCSVだけでは転換率もシート3も出せない）
