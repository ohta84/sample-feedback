# サンプルフィードバックExcel自動生成

Bカートの2種類のCSV（サンプル評価回収CSV・受注CSV）を受け取り、メーカー別のサンプルフィードバックExcelを自動生成するスキル。

## 必要なファイル

1. **サンプル評価回収CSV**（例: `サンプル評価回収.csv`）
   - 会社名・納品日・商品名・受注数・評価都合／採用数・サンプル評価結果 などを含む、評価データの本体
   - ただし**サンプル注文行と通常注文行の両方が混在している**ため、そのままでは使えない（後述）
2. **Bカート受注CSV**（例: `bcart_order.csv`）
   - 受注番号・商品名・業態・配送先都道府県・セット名など全項目を含む生データ
   - このCSVの**セット名**列（サンプル評価回収CSVには存在しない）で、サンプル注文か通常注文かを判定する
   - また、会社ごとに通常注文（転換）があったかどうかもこのCSV全体から判定する

両方をアップロードしてもらうこと。1つしか無い場合は不足しているものを聞く。

## 全体の流れ

1. ユーザーから上記2つのCSVを受け取る
2. **受注番号＋商品名**をキーに、評価CSVへ受注CSVから「業態」「配送先都道府県」「セット名」を結合する
3. セット名に「サンプル」を含む行だけを対象にする（評価CSVには通常注文行も混ざっているため必須）
4. 会社名に「ナゴミヤ」「なごみや」を含む行は自社テストのため除外
5. メーカーコード（商品名冒頭の `[121]` 等）で対象商品を絞り込む（任意）
6. メーカー名が未指定の場合、`data/maker_codes.csv`（コード→会社名の対応表）から自動解決する
7. `scripts/generate_feedback_excel.py` を実行してExcelを生成する
8. 生成後、社数・転換率などのサマリーをユーザーに提示し、内容に問題ないか確認する

## メーカーコード対応表（data/maker_codes.csv）

「数字, 会社名」の2列CSV。商品名冒頭の `[141]` のようなコードとメーカー名の対応表。
新しいメーカーコードが出てきたら、ユーザーに確認してこのファイルに追記していく。

`--maker-name` を省略して `--maker-code` だけ指定すると、この表から自動的に
`{コード}{会社名}` という形式（例: `141東京堂`）でタイトル・ファイル名が決まる。

## Excel生成ロジック（scripts/generate_feedback_excel.py に実装済み）

### シート1: サンプルフィードバック

サマリー（行1〜6）
- 行1: タイトル「(コード)(メーカー名)様サンプルフィードバック」太字・サイズ18・折り返しなし
- 行2: サンプル提供社数（ユニーク会社数）
- 行3: 注文転換社数（サンプル提供後に通常注文を1件以上行った会社数。受注CSV全体から判定）
- 行4: 注文転換率＝転換社数÷提供社数。赤字・太字・%表示
- 行5: 空白
- 行6: 左＝納品日の期間（最小〜最大）、右＝「なごみや太田」（更新者名・変更しないこと）

ヘッダー行（行7）とデータ（行8〜）

| 列 | 内容 | 幅 |
|---|---|---|
| A | 納品日（YYYY/MM/DD） | 14.6 |
| B | 業態（受注CSVから結合） | 15.1 |
| C | 会社名 | 31.6 |
| D | 配送先都道府県（受注CSVから結合） | 11.9 |
| E | 商品名 | 37.0 |
| F | サンプル用途（評価CSVの実データ） | 35.2 |
| G | サンプル評価結果（評価都合／採用数＋サンプル評価結果、評価CSVの実データ） | 71.4 |
| H | 注文有無 | 14.0 |

- ヘッダー行はグレー背景・中央寄せ・折り返しあり・枠線あり
- 同じ会社・同じ納品日の行が複数商品にまたがる場合、A,B,C,D,F,G,H 列をセル結合（E商品名のみ全行に入力）
- 注文有無：「✅ 注文あり」緑背景（E2EFDA）・緑文字（375623）・太字／「－ なし」グレー文字（999999）
- フォントは全体 Arial・サイズ11、枠線は薄いグレー（BFBFBF）細線

### シート2: 業態別分析

業態ごとに以下を集計する表（提供社数の多い順にソート）：
- 提供社数／転換社数／転換率
- OK件数・NG件数・検討中件数（評価コメント内の文字列一致による概算値。注記を必ず添える）

## 実行方法

```bash
pip install pandas openpyxl --break-system-packages
python scripts/generate_feedback_excel.py \
  --eval-csv "サンプル評価回収.csv" \
  --order-csv "bcart_order.csv" \
  --maker-code "141" \
  --output "141東京堂様サンプルフィードバック.xlsx"
```

`--maker-name` を明示すればそれを優先する。`--maker-code` は複数指定可（カンマ区切り）。

生成後は必ず `present_files` でユーザーに提示すること。

## 注意事項

- スクリプトの実行はClaude上で完結するので、ユーザー側のインストール作業は不要
- 「なごみや太田」の部分は作成者名なので絶対に変更しない
- 評価CSVには通常注文行も混ざっているため、必ず受注CSVのセット名で「サンプル」行に絞り込んでから集計すること（これを忘れると社数・転換率が過大に出る）
- CSVの実際のヘッダー名がBカートの標準と異なる場合があるため、スクリプトは列名を柔軟にキーワード一致で探す。うまく検出できない場合はエラーメッセージで該当箇所を出す
- 新しいメーカーコードが出てきたら `data/maker_codes.csv` に追記して更新する
