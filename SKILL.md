# サンプルフィードバックExcel自動生成

Bカート受注CSVを受け取り、メーカー別のサンプルフィードバックExcelを自動生成するスキル。

## 全体の流れ

1. ユーザーからBカート受注CSV（1ファイル）を受け取る
2. CSVの中身から**メーカー名**と**メーカーコード**（商品名冒頭の `[121]` 等）をユーザーに確認する
   - 確認できない場合は、CSV内の商品名一覧を提示してユーザーに選んでもらう
3. `scripts/generate_feedback_excel.py` を実行してExcelを生成する
4. 生成後、行数・社数などのサマリーをユーザーに提示し、内容に問題ないか確認する

## CSVの仕様

- Bカートの受注CSV。文字コードは Shift-JIS(CP932) の場合が多いが UTF-8 の場合もあるため自動判定する
- **BR列（69列目、0-indexで68）がセット名**。値に「サンプル」という文字列が含まれていればサンプル注文、含まれなければ通常注文として扱う
- 会社名に「ナゴミヤ」または「なごみや」を含む行は自社テストのため必ず除外する
- 商品名の冒頭 `[121]` のような角括弧内の数字がメーカーコード。メーカーコードが分かっている場合はそのコードで商品名を絞り込む

## Excel生成ロジック（scripts/generate_feedback_excel.py に実装済み）

### サマリー（行1〜6）
- 行1: タイトル「○○様サンプルフィードバック」太字・サイズ18・折り返しなし
- 行2: サンプル提供社数（ユニーク会社数）
- 行3: 注文転換社数（サンプル提供後に通常注文を1件以上行った会社数）
- 行4: 注文転換率＝転換社数÷提供社数。赤字・太字・%表示
- 行5: 空白
- 行6: 左＝納品日の期間（最小〜最大）、右＝「なごみや太田」（更新者名・変更しないこと）

### ヘッダー行（行7）とデータ（行8〜）
| 列 | 内容 | 幅 |
|---|---|---|
| A | 納品日（YYYY/MM/DD） | 14.6 |
| B | 業態 | 15.1 |
| C | 会社名 | 31.6 |
| D | 配送先都道府県 | 11.9 |
| E | 商品名 | 37.0 |
| F | サンプル用途（空欄・手入力用） | 35.2 |
| G | サンプル評価結果（空欄・手入力用） | 71.4 |
| H | 注文有無 | 14.0 |

- ヘッダー行はグレー背景・中央寄せ・折り返しあり・枠線あり
- 同じ会社・同じ納品日の行が複数商品にまたがる場合、A〜Dおよび F,G,H 列をセル結合する
- 注文有無：
  - 「✅ 注文あり」緑背景（E2EFDA）・緑文字（375623）・太字
  - 「－ なし」グレー文字（999999）
- フォントは全体 Arial・サイズ11、枠線は薄いグレー（BFBFBF）細線

## 実行方法

```bash
pip install openpyxl chardet --break-system-packages
python scripts/generate_feedback_excel.py \
  --csv /path/to/bcart_order.csv \
  --maker-name "○○" \
  --maker-code "121" \
  --output "/mnt/user-data/outputs/○○様サンプルフィードバック.xlsx"
```

`--maker-code` は複数指定可（カンマ区切り、例 `--maker-code "121,122"`）。省略した場合はCSV内の全行を対象にする（CSVが既に1メーカー分に絞られている場合はこれでよい）。

生成後は必ず `present_files` でユーザーに提示すること。

## 注意事項

- スクリプトの実行はClaude上で完結するので、ユーザー側のインストール作業は不要
- CSVを1ファイル投げてもらうだけでExcelが出てくる、という体験を崩さないこと
- 「なごみや太田」の部分は作成者名なので絶対に変更しない
- CSVの実際のヘッダー名がBカートの標準と異なる場合があるため、スクリプトは列名を柔軟にキーワード一致で探す。うまく検出できない場合はエラーメッセージで該当CSVのヘッダー一覧を出す
